# 问题分级标准

## 分级定义

| 级别 | 名称 | 定义 | 影响 |
|------|------|------|------|
| **P0** | 严重 | 数据质量问题 | 阻断上线 |
| **P1** | 高优 | 设计缺陷 | 需优先修复 |
| **P2** | 中优 | 风险提示 | 建议优化 |
| **P3** | 低优 | 代码风格 | 可选改进 |

---

## 评分机制

### 门禁（Pass/Fail）

- P0 问题数 > 0 => 结论 = **不通过**

### 质量分（quality_score）

- 初始 100 分，仅按 P1/P2/P3 扣分，最低 0 分
- P1: **-10**
- P2: **-3**
- P3: **-1**
- `-- 未评审` 项不计分

### 结论（三态）

| 条件 | 结论 |
|------|------|
| P0 > 0 | 不通过 |
| P0 = 0 且 quality_score >= 70 | 通过 |
| P0 = 0 且 quality_score < 70 | 有条件通过 |

---

## 各级别典型问题

### P0 严重

| 维度 | 问题 |
|------|------|
| 粒度主键 | 粒度不清晰/主键重复/隐藏扇形 |
| 分层引用 | 禁止跨层引用（DWS 引 ODS） |
| dbt 配置 | 配置完全缺失（无 description） |

### P1 高优

| 维度 | 问题 |
|------|------|
| 命名规范 | 表名前缀不符/字段命名错误 |
| 字段类型 | 日期用 STRING/金额用 DOUBLE |
| SCD 策略 | 策略与需求不匹配/缺少有效期字段 |
| 标准字段 | 事实表缺 is_deleted/etl_date |

### P2 中优

| 维度 | 问题 |
|------|------|
| 可加性 | 度量未标注可加性 |
| 分区键 | 分区键选择可优化 |
| dbt tests | 关键字段缺少 tests |

### P3 低优

| 维度 | 问题 |
|------|------|
| 命名风格 | 命名过长/缩写不一致 |
| 注释 | 注释表述可完善 |
| description | 非关键字段缺少 description |
